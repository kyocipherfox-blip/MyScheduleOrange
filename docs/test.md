# テスト仕様書 — 週間スケジュール表

## 1. テスト方針

本アプリはバックエンドを持たない静的 Web アプリケーションであり、
ブラウザ上での手動テストを基本とする。
テスト環境を揃えるため、以下の手順でブラウザを開いてテストを実施すること。

```
# GitHub Pages
https://kyocipherfox-blip.github.io/MyScheduleOrange/

# ローカル（file:// または開発サーバー）
open index.html
# または
npx serve .
```

## 2. テスト環境

| 項目 | 対象 |
|------|------|
| ブラウザ | Chrome 最新版・Firefox 最新版・Safari 最新版 |
| デバイス | PC（MacOS / Windows）・スマートフォン（iOS / Android） |
| 画面幅 | 360px / 768px / 1280px |

## 5. 既知の制限

| 制限 | 内容 |
|------|------|
| `file://` プロトコル | ES Modules は `file://` では CORS エラーとなる場合がある。ローカル開発時は `npx serve .` 等の簡易サーバーを使用すること |
| 複数端末同期 | データは端末ローカルに保存されるため、複数端末間での同期は不可 |
| タイムゾーン | 日付変換はローカルタイムを使用しているため、タイムゾーンをまたぐ利用は考慮外 |
